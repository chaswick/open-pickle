# Trademark And Branding Policy

This is a lightweight pre-release branding policy stub for the PickleBuddies project.

## Scope

The software license for this repository does not automatically grant rights to use the project name, logos, or branding as if they were official.

This file is meant to set the default expectation until a more formal policy exists.

## Allowed Uses

You may:

- refer to PickleBuddies by name when truthfully describing the project
- say that your fork, integration, or deployment is “based on PickleBuddies” or “compatible with PickleBuddies”
- link to the project and discuss it publicly

## Uses That Need Different Branding

If you operate a modified fork, separate hosted service, or derivative distribution, you should use distinct branding if:

- you changed behavior in a way that users could reasonably confuse with the official project
- you run the software as an independent public service
- you distribute a modified version under your own maintenance

In those cases, keep factual attribution, but do not present your fork or service as the official PickleBuddies project.

## No Implied Endorsement

Do not use the PickleBuddies name, logos, or branding in a way that implies:

- official sponsorship
- endorsement by the maintainer
- that your hosted service is the canonical project service when it is not

## Project Assets

Until a fuller brand policy is published, treat the project name and visual identity as project branding rather than as assets automatically granted for unrestricted reuse.

## Future Revision

This file is an initial policy stub and may be replaced with a more formal trademark and logo policy before or after the public launch.
