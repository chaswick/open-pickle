# Security Policy

## Status

This repository is in pre-public-release preparation.

## Reporting A Vulnerability

Do not open a public issue for a suspected security problem.

Until a dedicated public security contact is published, report vulnerabilities privately to the maintainer through the existing private contact channel already used for this project.

## What To Include

Please include:

- a short description of the issue
- affected route, page, or component
- reproduction steps
- impact assessment if known
- any proof-of-concept details needed to reproduce safely

## Supported Versions

At this stage, only the current mainline codebase should be considered for fixes.

## Pre-Launch Note

Before the repository is made public, replace the reporting section above with a monitored security contact address or private disclosure mechanism.
