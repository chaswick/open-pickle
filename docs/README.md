# Docs Index

**Status:** Active reference

## Purpose

Keep the top-level `docs/` folder limited to documents that describe the current public repo, current codebase, or current release posture.

## Freshness Rule

- Top-level `docs/` should describe the codebase as it exists now.
- If a document is mainly useful as background, design history, or an earlier proposal, keep it out of the public repo unless it still helps current contributors.
- If a document has not been reviewed recently, treat it as stale until it is refreshed or removed.
